一、应用简介

Minecraft 多服务器模组管理器 是一款面向 Minecraft 服务器管理员的桌面工具， 基于 Electron + Vue 3 构建。它帮助你统一管理多个服务器的模组文件， 通过 Modrinth API 检测模组更新，并通过"公共模板"机制在服务器之间同步模组配置。

模组管理

扫描服务器 mods 目录、查看版本、启用/禁用、批量操作、检查更新

更新检测

通过 Modrinth 官方 API 自动识别模组并比对最新版本

模板同步

以公共模板为基准，在多服务器间同步模组，确保一致性

多服务器

同时管理多个 Minecraft 服务器，独立配置、快速切换

深色模式

支持浅色/深色主题切换，适应不同使用环境

便携设计

提供安装版和便携版，无需复杂配置即可使用

适用场景：适用于任何使用 Modrinth 平台模组的 Minecraft 服务器（Paper、Fabric、NeoForge、Spigot 等）。典型的用户是同时维护多个 Velocity/Fabric 服务器集群的管理员。

二、快速开始

添加服务器 第一步

  1. 启动程序，点击左侧菜单 设置 进入设置页面
  2. 在"服务器管理"标签页，点击 添加服务器 按钮
  3. 输入服务器名称（如"生存服"、"创造服"）
  4. 点击路径选择框，选择服务器根目录（应包含 mods 子目录）
  5. 点击确认，服务器即被添加
提示：添加成功后，左侧菜单的"服务器"子菜单会自动刷新，出现你刚添加的服务器条目。

扫描模组 第二步

  1. 点击左侧菜单的服务器名称，进入服务器概览页
  2. 点击 管理模组 按钮，进入模组管理页面
  3. 程序会自动扫描 mods/ 目录，读取所有 .jar 文件
  4. 扫描完成后，模组以表格形式展示，包含文件名、启用状态、版本号等信息
说明：被禁用的模组文件以 .jar.disabled 后缀标识。程序会将其显示为"禁用"状态，并在文件名上添加删除线样式。

三、界面导航

应用界面采用经典的左侧菜单 + 右侧内容区布局：

  • 左侧菜单栏 — 包含公共模板、服务器（子菜单）、同步管理、设置四个入口。菜单栏可点击折叠按钮收缩为图标模式，为内容区留出更多空间。
  • 顶部标题栏 — 显示应用名称，右侧提供深色模式切换按钮。
  • 底部状态栏 — 显示全局统计信息：模组总数、待更新数量、同步状态。
  • 内容区 — 展示当前页面的具体功能界面。

四、模组管理

模组管理是本应用的核心功能。在服务器概览页点击"管理模组"即可进入。

扫描模组

  • 进入模组管理页面时，程序会自动扫描当前服务器的 mods/ 目录
  • 你也可以随时点击 重新扫描 按钮手动刷新模组列表
  • 扫描结果包含：文件名、状态（启用/禁用）、版本号（从文件名自动提取）
  • 支持 .jar 和 .jar.disabled 两种后缀的文件

检查更新

  1. 点击 检查更新 按钮，程序将逐个查询 Modrinth API
  2. 进度条显示当前处理进度（正在处理：当前数 / 总数）
  3. 比对完成后，有更新的模组会在"更新"列显示红色 有更新 标签
  4. 底部状态栏会更新待更新数量
注意：更新检测依赖 Modrinth 平台。若模组未发布在 Modrinth 上，或文件名与 Modrinth 项目名差异过大，可能无法识别。每次 API 请求间隔约 1 秒以避免触发限流，因此检查 27 个模组约需 30 秒。

启用 / 禁用

  • 单个操作：在模组表格的"操作"列，点击"启用"或"禁用"按钮
  • 实现方式：启用/禁用通过重命名文件实现——禁用时添加 .disabled 后缀，启用时去除该后缀。不会修改模组文件内容，可随时切换

删除模组

  • 点击操作列的 删除 按钮
  • 程序会弹出确认对话框，确认后从磁盘永久删除该文件

批量操作

模组表格支持多选，选中后会显示批量操作栏：

  1. 勾选表格左侧的复选框，选中一个或多个模组
  2. 顶部出现批量操作栏，显示"已选中 N 个模组"
  3. 可执行：批量启用、批量禁用、批量删除
  4. 点击 取消选择 清除所有选中状态

右键菜单

在任意模组行上右键点击，弹出快捷操作菜单：

  • 启用 / 禁用 — 根据模组当前状态显示对应操作
  • 删除 — 确认后删除文件
  • 打开所在文件夹 — 在系统资源管理器中定位该文件

点击菜单外部或再次右键可关闭菜单。

小技巧：在模组文件名行悬停，可查看更多文件名内容（超出部分自动省略号截断）。在右键菜单中选择"打开所在文件夹"是最快捷的定位方式。

五、同步管理

同步管理是本应用的核心功能之一。它基于"公共模板"机制工作： 你建立一个模板目录作为基准，各个服务器与模板进行差异对比和同步， 确保所有服务器的模组保持一致。

前置条件：使用同步功能前，请先在设置页面配置公共模板路径，并至少添加一个服务器。

扫描差异

  1. 点击左侧菜单 同步管理 进入同步页面
  2. 页面以服务器分组展示，每个服务器独立显示差异状态
  3. 点击 扫描差异 按钮开始比对模板与服务器的文件
  4. 比对基于 SHA256 哈希值，确保内容精确匹配

差异类型有三种：

  • 新增 — 模板目录中有，但服务器缺少的文件，需要同步到服务器
  • 服务器独有 — 服务器有但模板没有的文件，可以收集回模板
  • 已修改 — 模板和服务器都存在但内容不同（哈希值不匹配）

执行同步

  • 在差异树中勾选需要同步的文件
  • 点击 同步选中 仅同步勾选的文件
  • 点击 同步全部 同步所有差异文件

忽略管理

  • 在差异树中勾选文件后，点击 忽略选中 可将文件加入忽略列表
  • 被忽略的文件将不再显示在差异结果中
  • 展开 已忽略的项目 可查看所有被忽略的文件并取消忽略
  • 忽略配置存储在本地数据库中，重启后依然有效
安全机制：同步引擎会自动排除系统关键文件，如 server.properties、eula.txt、ops.json、whitelist.json、bukkit.yml、spigot.yml、paper.yml 等，以及 .git、libraries、versions、logs、cache 等目录。这防止了关键配置被意外覆盖。

六、公共模板

公共模板 是一个参考目录，作为所有服务器模组配置的基准。 你可以在模板中放置所有服务器都应该拥有的通用模组。

  • 点击左侧菜单 公共模板 查看模板目录的文件树
  • 文件树展示目录结构和文件的修改时间、大小等元信息
  • 模板路径在设置页面配置
设计理念：配置继承链遵循 全局默认配置 → 公共模板 → 服务器特化 的层级。公共模板保存通用配置，每个服务器在同步后可以再按需调整特化配置。

七、设置

设置页面提供服务器和模板路径的管理功能。

服务器管理

  • 添加服务器 — 填写服务器名称，选择服务器根目录（目录下需包含 mods 子目录才有效）
  • 编辑名称 — 在服务器卡片上直接点击名称即可内联编辑
  • 删除服务器 — 从数据库中删除服务器记录，不会删除服务器文件
  • 统计信息 — 每个服务器卡片展示模组总数、启用数和禁用数

模板路径

  • 在"公共模板库"标签页中，点击选择目录按钮
  • 选择你想要作为模板的目录
  • 配置后，公共模板页面和同步功能即可使用

九、深色模式

应用支持浅色和深色两种主题，适应不同的使用环境。

  • 切换方式：点击顶部标题栏右侧的主题切换按钮（太阳/月亮图标）
  • 状态持久化：主题选择会自动保存，下次启动应用时保持上次的选择
  • 即时生效：切换无需重启，所有界面元素同步切换

十、常见问题

Q: 为什么扫描不到某些模组？

程序仅识别 mods/ 目录下以 .jar 或 .jar.disabled 结尾的文件。如果模组在其他子目录中（如 plugins/），请确保它位于 mods/ 目录下。

Q: 为什么检查更新显示"0 个待更新"？

可能的原因：

  • 模组文件名与 Modrinth 上的项目名差异过大，未能识别
  • 模组未发布在 Modrinth 平台
  • 本地版本号与 Modrinth 最新版本号一致（确实无需更新）
  • 网络连接问题导致 API 请求失败

Q: 启用/禁用模组会影响服务器吗？

不会影响服务器运行。启用/禁用通过修改文件后缀实现（.jar ↔ .jar.disabled），不修改文件内容。禁用后的模组不会被服务器加载，启用后恢复正常。你可以在服务器运行时操作，但部分服务端可能需要重启才能生效。

Q: 同步功能会覆盖我的配置文件吗？

不会。同步引擎内置了安全排除列表，会自动跳过 server.properties、eula.txt、ops.json、whitelist.json、bukkit.yml、spigot.yml、paper.yml、commands.yml、permissions.yml、help.yml 等关键配置文件，以及 .git、libraries、versions、logs、cache 等目录。

Q: 数据存储在哪里？

所有应用数据（服务器列表、配置、同步忽略列表等）存储在 SQLite 数据库中，数据库文件位于系统用户数据目录下的 mod-manager.db。具体路径取决于操作系统：

  • Windows: %APPDATA%/Minecraft 模组管理器/mod-manager.db

Q: 如何备份我的数据？

直接备份上述路径中的 mod-manager.db 文件即可。该文件包含了所有服务器配置和忽略列表。

Q: 程序运行卡顿怎么办？

程序已针对性能做了多项优化（GPU 加速、渲染节流、响应式优化等）。如果仍有卡顿，可以尝试：

  • 确保显卡驱动已更新到最新版本
  • 检查是否同时打开了大量其他程序占用 GPU 资源
  • 在设置中减少同时管理的服务器数量
  • 尝试重启程序

Q: 如何卸载？

安装版：通过 Windows 的"程序和功能"卸载。便携版：直接删除程序文件即可。卸载后如需清理数据，可手动删除 %APPDATA%/Minecraft 模组管理器 目录。

十一、技术说明

以下信息供对技术细节感兴趣的进阶用户参考。

技术栈

  • 前端框架：Vue 3（Composition API）+ Element Plus UI 库
  • 状态管理：Pinia
  • 桌面框架：Electron 42
  • 构建工具：Vite 8 + Rolldown
  • 数据存储：sql.js（SQLite WebAssembly 实现）

数据安全

  • 所有数据存储在本地，不上传任何服务器
  • Modrinth API 调用仅用于搜索模组信息和版本比对，不传输服务器文件内容
  • 文件操作（删除、重命名等）均在本地磁盘执行

更新检测原理

  1. 从文件名中提取模组名（去除版本号后缀，如 fabric-api-0.70.0+1.19.2.jar → fabric-api）
  2. 通过 Modrinth API 搜索该名称，匹配最佳结果
  3. 获取该模组在 Modrinth 上的最新版本号
  4. 与本地文件名中提取的版本号进行字符串比对
  5. 若不一致则标记为"有更新"

同步原理

  • 递归遍历模板目录和服务器的 mods/ 目录
  • 对每个文件计算 SHA256 哈希值
  • 根据哈希值配对判断"新增"、"已修改"、"服务器独有"三种差异类型
  • 同步操作本质是文件复制（新增/修改方向：模板 → 服务器；独有方向：服务器 → 模板）
